# YouTube Blog Pipeline

Автоматический парсер YouTube-каналов + генератор SEO/GEO-оптимизированных статей для блога [Синапс](https://synapse.blog).

## Что делает

1. **Парсит RSS** 14 YouTube-каналов (AI, стартапы, маркетинг)
2. **Скачивает транскрипции** через youtube-transcript-api
3. **Генерирует экспертные статьи** через Claude API (Sonnet)
4. **Выводит готовый MDX** — можно сразу публиковать в Astro-блог

## Архитектура

```
config.py           → Список каналов, API ключи
parser_rss.py       → Парсинг RSS-фидов YouTube
transcriber.py      → Скачивание субтитров
article_generator.py → Системный промпт + генерация через Claude API
pipeline.py         → Оркестратор: парсинг → транскрипция → генерация → публикация
```

## Быстрый старт

```bash
# Зависимости
pip install -r requirements.txt

# Настройка
export ANTHROPIC_API_KEY="sk-..."

# Запуск (5 последних видео)
python pipeline.py --limit 5

# Запуск + автопубликация в блог
python pipeline.py --limit 5 --publish
```

## Принципы написания статей (системный промпт)

Главный актив пайплайна — промпт в `article_generator.py`. Вот ключевые принципы:

### Роль
Главный редактор технологического медиа. Аудитория: предприниматели, маркетологи, продакт-менеджеры. Не разработчики, но технически грамотные.

### Ключевой принцип
**НЕ пересказ, а самостоятельная экспертная статья.** Транскрипт — основа, но модель обязана дополнять из своих знаний:
- Идея на 2 предложения → полноценный раздел с контекстом и примерами
- Упоминается инструмент без объяснения → объясни
- Есть цифры → дай контекст (рост/падение, сравнение)
- Добавляй actionable-шаги, которые читатель может выполнить сегодня

### Запреты
- Никаких упоминаний видео/канала/спикера/подкаста — статья = оригинальный материал редакции
- Без канцелярита: "следует отметить", "давайте рассмотрим", "стоит подчеркнуть"
- Без воды: "в современном мире", "не секрет что", "все мы знаем"
- Не выдумывать статистику — лучше обобщить ("по данным индустрии")

### Структура
- **H1**: до 70 символов, содержит поисковый запрос, конкретный
- **Лид**: хук → суть → ключевое слово, короткие предложения
- **Тело**: 5-8 разделов H2, каждый 200-400 слов (самодостаточный блок для GEO)
- **Практика**: минимум 1 пошаговый план + 1 раздел оценки (плюсы/минусы)
- **FAQ**: 4-6 вопросов как в "Люди также спрашивают", ответы 2-4 предложения

### SEO-оптимизация
- Ключевое слово в H1, первых 100 словах, 2-3 подзаголовках
- Плотность 1-2%, естественные вхождения
- 5-8 LSI-ключевых слов рассыпаны по тексту
- Meta title до 60 символов, meta description 150-160 символов с CTA

### GEO-оптимизация (Generative Engine Optimization)
Оптимизация для AI-поисковиков (Perplexity, Google AI Overviews, ChatGPT Search):
1. **Самодостаточные секции** — каждый H2 = законченный ответ, можно процитировать без контекста
2. **Каноничные ответы** — первые 1-2 предложения раздела напрямую отвечают на вопрос подзаголовка
3. **Структурированные данные** — списки, таблицы, шаги (AI предпочитает структуру)
4. **Авторитетные формулировки** — "По состоянию на 2026 год...", "На практике это означает..."
5. **Числовая конкретика** — не "дорого", а "$200/мес"; не "быстро", а "за 15 минут"
6. **Сравнения** — если инструмент, то кратко vs 1-2 альтернативы

### Формат вывода
JSON с полями: `meta_title`, `meta_description`, `slug`, `keywords`, `tags`, `h1`, `excerpt`, `content_mdx`, `faq`

Контент в чистом Markdown (MDX), минимум 1500 слов. FAQ отдельным массивом для структурированной разметки (FAQPage schema).
